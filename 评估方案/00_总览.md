## 2. 主体分析链路

当前项目的主链路由以下模块组成：

1. `app/services/openface_service.py`
   - 运行 OpenFace。
   - 输出包含 `timestamp`、`frame`、`face_id`、`gaze_angle_x/y`、AU 强度等字段的 CSV。

2. `app/services/frame_service.py`
   - 读取 OpenFace CSV。
   - 多人脸时自动锁定主体脸。
   - 基于 OpenFace 时序直接计算：
     - 视线注意力
     - 微表情 / AU 异常事件

3. `app/services/pipeline_service.py`
   - 按时间轴抽帧。
   - 对“宏表情采样帧”调用：
     - `DeepFaceService.analyze()`：宏表情分类分布
     - `EmoNetService.analyze()`：VA 分析
   - 最终将各模块结果汇总到 summary。

4. `app/services/rppg_service.py`
   - rPPG 与呼吸分析是独立分析链路。
   - 通过 API 手动触发，不在主视频分析链路中自动运行。

## 3. 五类方案与代码对应关系

### 3.1 宏表情

- 汇总入口：`app/services/pipeline_service.py`
- 模型封装：`app/services/deepface_service.py`
- 实际后端：
  - `app/services/dan_backend.py`
  - `app/services/affectnet_backend.py`

注意：虽然字段名仍沿用 `deepface_*`，但默认后端不一定真的是 DeepFace。根据 `app/config.py`，`SG_DEEPFACE_BACKEND` 默认是 `dan`。

### 3.2 宏表情 VA

- 汇总入口：`app/services/pipeline_service.py`
- 服务封装：`app/services/emonet_service.py`
- 实际优先后端：`app/services/affectnet_backend.py`
- 回退后端：`app/services/dan_backend.py`

注意：虽然字段名叫 `emonet_*`，但当前实现实际优先走 AffectNet 回归输出，并非旧版 EmoNet 主体推理。

### 3.3 微表情 AU

- 核心实现：`app/services/frame_service.py`
- 主流程当前实际使用：`detect_abnormal_events()`
- 项目中仍保留旧方法：`detect_micro_expression_events()`，但未接入主汇总流程。

### 3.4 视线注意力

- 核心实现：`app/services/frame_service.py`
- 主流程汇总：`app/services/pipeline_service.py`
- 输入来自 OpenFace 的：
  - `gaze_angle_x`
  - `gaze_angle_y`
  - `success`
  - `confidence`
  - `AU45_c`
  - `AU45_r`

### 3.5 rPPG / 呼吸

- 核心实现：`app/services/rppg_service.py`
- API 触发入口：`app/blueprints/api_tasks.py`

## 4. 代码优先时需要特别注意的命名问题

### 4.1 `deepface_*` 不等于真正的 DeepFace

`app/services/deepface_service.py` 只是保留旧接口名，底层实际可切换到：

- DAN
- AffectNet

默认配置下：

$$
\text{backend}_{macro} = \text{dan}
$$

### 4.2 `emonet_*` 不等于旧版 EmoNet 模型

`app/services/emonet_service.py` 当前逻辑是：

$$
\text{primary} = \text{AffectNet regression},\quad
\text{fallback} = \text{DAN prior-based VA estimate}
$$

### 4.3 微表情实际输出是 AU 异常事件

主流程 summary 中当前返回的是：

- `abnormal_events`

而不是旧版的 AU 突增清单。

## 6. 总结

从实现层面看，项目实际上形成了“两条分析链”：

- 主链：OpenFace -> 主体脸 -> 抽帧 -> 宏表情 / VA -> AU异常事件 / 视线注意力 -> summary
- 辅链：指定时间段视频 -> RhythmMamba -> rPPG -> 呼吸与深呼吸合规性

因此，后续如果要做论文、汇报或产品说明，最稳妥的表达方式不是直接复述旧文档，而是以当前代码中真实的输入、阈值、评分函数、输出字段为准。
