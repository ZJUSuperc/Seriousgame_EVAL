# AU 异常事件检测新方案（纯 AU）

目标：

- 做“异常事件检测”而不是整体情绪分析。
- 输出：异常类型、发生时间、持续时长、触发的 AU、强度与变化。
- 最终结果**覆盖现有微表情事件列表**，以“异常事件列表”替代。

---

## 一、异常的统一定义（核心）

一个“异常事件”= 显著偏离中性状态 + 持续时间满足要求 +（可选）突发变化。

三要素：

```
强度（Intensity）
持续时间（Duration）
变化率（Delta）
```

---

## 二、时间规则（结合 OpenFace 时间轴）

OpenFace 帧率约 30 fps，且提供 timestamp：

- **优先使用 timestamp** 计算持续时长。
- 若 timestamp 缺失，则用帧数换算：

```
0.3s ≈ 9 帧
0.5s ≈ 15 帧
1.0s ≈ 30 帧
```

建议最小持续时间：**≥ 0.5s**。

---

## 三、事件类型与判别规则（不统一阈值）

以下每个事件都有独立阈值与规则。

### ① 长闭眼（Fatigue / 走神）

判定：

```
AU45_r > 0.5 持续 ≥ 1.0s
```

输出：

```
long_eye_closure_event
```

---

### ② 高兴（Positive Event）

判定（推荐）：

```
AU12_r ≥ 0.55 且 AU6_r ≥ 0.4
持续 ≥ 0.5s
```

或（强微笑兜底）：

```
AU12_r ≥ 0.85 持续 ≥ 0.5s
```

输出：

```
positive_event
```

---

### ③ 难过（Negative Low Arousal）

判定：

```
AU15_r > 0.5 持续 ≥ 0.5s
```

输出：

```
sad_event
```

---

### ④ 厌恶 / 不适（Aversion）

判定：

```
AU9_r > 0.6 或 AU10_r > 0.6
```

替代（更稳定）：

```
AU14_r > 0.5 且 AU15_r > 0.5
```

输出：

```
aversion_event
```

---

### ⑤ 紧张 / 压力（Stress）

判定（核心）：

```
AU4_r > 0.5 且 AU7_r > 0.5
```

输出：

```
stress_event
```

---

### ⑥ 突发反应（Surprise-like）

判定（变化率）：

```
ΔAU1 或 ΔAU2 或 ΔAU5 > 0.5
且 AU1_r、AU2_r、AU5_r 同时 > 0.5
```

输出：

```
surprise_event
```

---

### ⑦ 困惑（Confusion）

判定：

```
AU4_r > 0.5 且 AU1_r > 0.5
```

输出：

```
confusion_event
```

---

## 四、变化率检测（用于突发事件）

定义：

```
ΔAU = |AU_t - AU_(t-1)|
```

推荐阈值：

```
ΔAU > 0.5
```

用途：用于 `surprise_event` 与突发反应类事件的触发。

---

## 五、统一检测流程（工程实现）

### Step 1：逐帧计算

```
读取 AU_r
计算 ΔAU
```

### Step 2：触发条件

```
按事件规则判定每帧的候选状态
```

### Step 3：事件生成

```
连续帧合并为一个事件
```

### Step 4：过滤

```
duration < 0.5s → 丢弃
```

---

## 六、异常强度（建议）

```
intensity = 事件相关 AU 的峰值
```

---

## 七、事件输出结构（覆盖微表情事件列表）

### 事件列表（summary.json 字段：abnormal_events）

```json
[
  {
    "type": "stress_event",
    "start": 12.3,
    "end": 13.8,
    "duration": 1.5,
    "intensity": 0.9,
    "au_list": ["AU4", "AU7"],
    "au_peak": {"AU4": 0.9, "AU7": 0.8}
  },
  {
    "type": "long_eye_closure_event",
    "start": 25.8,
    "end": 26.7,
    "duration": 0.9,
    "au_list": ["AU45"],
    "au_peak": {"AU45": 1.2}
  }
]
```

### 时间轴标记（UI用）

```
12s   → 紧张
26s   → 长闭眼
35s   → 高兴
```

---

## 八、前端展示与跳转规则

- **替换微表情事件列表**：展示异常事件类型、触发 AU、强度与时间。
- **支持点击跳转**：点击事件跳到对应时间点。
- **若当前帧不可显示**（非宏表情采样帧），自动跳到**最近可显示帧**。

---

## 九、关键设计原则（必须遵守）

```
不要用单一 AU 直接做情绪判断
必须采用 AU 组合或变化率
连续时间段 → 事件
```

---

## 一句话总结

```
这是一个“异常行为时间定位系统”
输出：什么时候、发生了什么异常、持续多久、强度如何
```
