很好，这样问题就变得**非常清晰**了：

* 你 **已经有 VA（Valence-Arousal）分析**
* 现在 **只想针对宏表情（macro-expression）做一个独立分析模块**
* 目标是 **应用场景：老年人严肃游戏状态评估**

所以这个模块的目标就不应该是：

```
识别情绪类别
```

而应该是：

```
分析宏表情的行为模式
```

也就是：

> **宏表情在任务过程中出现了什么结构特征**

我给你设计一套 **更适合工程应用 + 可解释 + 与心理状态相关的宏表情分析框架**。

整体只做 **4件事**：

```
宏表情结构
宏表情频率
宏表情稳定性
宏表情强度
```

---

# 一、宏表情分析模块整体结构

```
宏表情帧序列
↓
表情概率平滑
↓
宏表情事件检测
↓
宏表情统计
↓
宏表情评分
```

输出：

```
macro_emotion_profile
macro_emotion_score
macro_emotion_stability
macro_emotion_activity
```

---

# 二、Step1：宏表情帧筛选

因为你是 **固定帧采样**，所以每一帧会有：

```
emotion probability
```

例如：

```
happy 0.55
neutral 0.30
sad 0.10
fear 0.03
anger 0.02
```

先做两个简单处理：

### 1 平滑

减少模型抖动：

```
P_t = 0.3 * current + 0.7 * previous
```

---

### 2 模糊帧过滤

如果：

```
最大概率 < 0.45
```

或者：

```
最大概率 - 第二大概率 < 0.1
```

标记为：

```
mixed
```

不参与宏表情事件。

---

# 三、Step2：宏表情事件检测（核心）

宏表情不是单帧，而是：

```
持续的情绪状态
```

例如：

```
happy
happy
happy
neutral
neutral
sad
sad
sad
```

我们把连续相同情绪合并：

```
happy (3帧)
neutral (2帧)
sad (3帧)
```

然后设置 **最小持续时间**：

例如：

```
≥2帧
```

才算：

```
macro emotion event
```

例如：

```
happy_event
duration = 3s
```

---

# 四、Step3：宏表情统计

对所有事件统计。

---

## 1 宏表情分布

```
macro_emotion_distribution
```

例如：

```
neutral 55%
happy 25%
sad 10%
surprise 6%
anger 3%
fear 1%
```

表示：

**整体情绪结构**

---

## 2 宏表情频率

统计：

```
emotion_event_count
```

例如：

```
happy events = 4
sad events = 2
```

然后计算：

```
events_per_minute
```

反映：

```
情绪变化活跃度
```

---

## 3 宏表情持续时间

计算：

```
mean_duration
max_duration
```

例如：

```
happy_mean_duration = 5s
sad_mean_duration = 3s
```

这个指标其实很重要：

因为：

```
负面情绪持续时间
```

比出现次数更重要。

---

## 4 宏表情稳定性

计算：

```
emotion_switch_rate
```

例如：

```
happy → neutral
neutral → sad
```

统计：

```
每分钟切换次数
```

说明：

```
情绪稳定 or 波动
```

---

## 5 表情活跃度

计算：

```
emotion_variance
```

即：

```
情绪概率变化程度
```

如果：

```
变化很小
```

说明：

```
表情非常少
```

可能：

```
低参与
抑郁倾向
```

---

# 五、Step4：宏表情评分

最终给出 **三个指标**。

---

# 1 情绪基调（Tone）

计算：

```
positive = happy + surprise + 0.5*neutral
negative = sad + fear + anger + disgust
```

然后：

```
tone = positive - negative
```

映射到：

```
0 - 100
```

含义：

```
情绪整体偏正 or 偏负
```

---

# 2 情绪稳定度

根据：

```
emotion_switch_rate
```

计算：

```
stability_score
```

例如：

```
switch_rate 低 → 稳定
switch_rate 高 → 波动
```

---

# 3 表情活跃度

根据：

```
emotion_event_count
emotion_variance
```

计算：

```
activity_score
```

例如：

```
表情很多 → 活跃
几乎没表情 → 低参与
```

---

# 六、最终宏表情评分

综合：

```
macro_emotion_score =
0.4 * tone_score
0.3 * stability_score
0.3 * activity_score
```

输出：

```
0 - 100
```

---

# 七、最终输出结构

建议输出：

```
macro_emotion_analysis
```

内容：

```
emotion_distribution
emotion_event_count
emotion_switch_rate
emotion_mean_duration

tone_score
stability_score
activity_score

macro_emotion_score
macro_emotion_rating
```

评级：

```
A ≥ 85
B ≥ 70
C ≥ 55
D < 55
```
